# Prompt 01 — Survey Repo

**Input:** `--repo <path>`
**Output:** `<out>/.scratch/repo-inventory.json`

## Preflight (run before anything else)

Confirm this is a Lovable project. Check **all four** fingerprints; need ≥2 to proceed:

1. `package.json` `devDependencies` contains `lovable-tagger`.
2. `vite.config.ts` (or `.js`) calls `componentTagger()`.
3. Directory `.lovable/` exists at repo root.
4. `src/integrations/supabase/client.ts` (or equivalent) starts with a "This file is automatically generated" comment.

Also acceptable hints: `.lovable/plan.md`, `components.json` (shadcn), template name `vite_react_shadcn_ts` in package.json.

If <2 hints: stop. Tell user "doesn't look like a Lovable project — aborting" and list what you checked.

## How

- Use Glob/Grep, not full repo reads.
- Routes: find `<Route>` elements or equivalent router config.
- Components: glob `src/components/**/*.{tsx,jsx,vue}`; exclude anything under `components/ui/`.
- Migrations: glob `supabase/migrations/*.sql`.
- Edge functions: glob `supabase/functions/*/index.ts`.
- Tailwind tokens: parse `tailwind.config.*` and any CSS variables in `src/index.css` / `src/App.css`.
- `.lovable/plan.md` history: `git -C <repo> log --follow --format="%H %s" -- .lovable/plan.md`.
- Keep per-file LOC only (don't dump file contents).

Scope guard: if inventory would exceed ~200KB JSON, summarise long arrays with a `_truncated_count` field and keep the top N most-referenced items.
